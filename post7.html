Post 7: Primary Key, Foreign Key, dan Candidate Key

Diposting pada: 16 Oktober 2025 | Kategori: Integritas Data

Konsep **Kunci (Key)** adalah inti dari Model Data Relasional. Kunci berfungsi untuk memastikan integritas data dan mendefinisikan hubungan antar tabel. Tanpa kunci yang tepat, database akan menjadi tidak konsisten.

1. Candidate Key (Kunci Kandidat)

**Candidate Key** adalah satu atau sekumpulan atribut minimum yang dapat mengidentifikasi secara unik sebuah baris dalam tabel. Setiap Kunci Kandidat harus **unik** dan **minimal**.

Primary Key (Kunci Utama)

**Primary Key (PK)** adalah Kunci Kandidat yang **dipilih** oleh perancang database untuk menjadi identitas unik utama sebuah tabel. PK tidak boleh memiliki nilai **NULL** (Integritas Entitas) dan harus unik.

**Contoh (Tabel GURU):** Candidate Keys bisa berupa **NIP** atau **(Nama_Guru + Tgl_Lahir)**. Kita memilih **NIP** sebagai Primary Key karena lebih pendek dan stabil.

Ilustrasi DDL Primary Key

-- NIP dijadikan Primary Key untuk identitas unik Guru
CREATE TABLE Guru (
    NIP CHAR(10) PRIMARY KEY,
    Nama_Guru VARCHAR(100) NOT NULL,
    Tgl_Lahir DATE 
);

2. Foreign Key (Kunci Tamu)

**Foreign Key (FK)** adalah atribut pada satu tabel (tabel anak) yang nilainya merujuk pada **Primary Key** di tabel lain (tabel induk). FK adalah jembatan yang menciptakan relasi dan menjamin **Integritas Referensial**.

**Contoh (Tabel SISWA merujuk ke KELAS):** Atribut **ID_Kelas** di tabel SISWA adalah Foreign Key yang merujuk pada Primary Key **ID_Kelas** di tabel KELAS.

Ilustrasi DDL Foreign Key

-- Tabel Siswa (Anak)
CREATE TABLE Siswa (
    NIS CHAR(10) PRIMARY KEY,
    Nama_Siswa VARCHAR(100),
    ID_Kelas CHAR(5),
    -- Definisi Foreign Key (FK) yang merujuk ke tabel KELAS
    FOREIGN KEY (ID_Kelas) REFERENCES Kelas(ID_Kelas)
);
